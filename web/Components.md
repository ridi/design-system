# Components

리디북스 서점, 리디 셀렉트, 페이퍼샵에서 공통으로 사용하는 UI 컴포넌트입니다.

## 버튼 Button 

- 버튼은 사용자가 특정 작업을 실행하거나 상태를 변경할 수 있도록 돕는 역할을 합니다.
- 사용자가 클릭하거나 터치할 때 발생하는 동작에 대한 피드백을 제공하고, 버튼 내 레이블은 사용자가 명확하게 이해할 수 있도록 직관적이고 간결하게 작성합니다. 
- Hover와 Active 상태를 통해 버튼이 활성화될 수 있음을 나타냅니다.
- Disabled 상태는 사용자가 실수하는 것을 미리 방지하거나 정보가 충분하지 않음을 알립니다.

### 크기

Height 값 기준으로 고정되어 있는 크기를 상황에 맞게 사용합니다. 버튼의 Width는 패딩으로 지정하고 필요한 경우 크기를 강제할 수 있습니다.

- 21 / 25 / 30 / 40 / 50

### 사용

#### **Text Button**

- 텍스트 버튼을 이용하여 사용자를 특정 페이지로 안내하거나 특정 작업을 실행할 수 있도록 돕습니다. 이때, 사용자가 누를 수 있는 버튼인지 인지할 수 있도록 반드시 Hover 피드백을 제공합니다. 
- 레이블 만으로 의미 전달이 어려울 경우에는 아이콘과 함께 사용하기도 합니다.

#### **Icon & Label Button**

- 버튼의 의미가 명확하지 않은 경우나 버튼의 의미를 강조하는 경우에 사용합니다.  

#### **Icon Only Button**

- 정해진 Height 값에서 정사각형의 형태를 가집니다. 아이콘의 모양에 따라 Width 값은 조정될 수 있습니다. 
- 아이콘 만으로 쉽게 인지가 되어야 하므로 의미를 명확하게 전달할 수 있는 형태를 사용하는 것이 중요합니다.

#### **Toggle Button**

- 두 가지 옵션을 그룹화하여 어떤 옵션이 활성화 되었는지 사용자에게 명확하게 알려줍니다. 옵션 값이 2개일 때 사용할 수 있습니다. 

### 그룹

기능이 다른 버튼을 같이 배치할 때, 버튼 크기와 사이 간격에 유의합니다. 또한 버튼 그룹에서 Primary Button을 우측 또는 상단에 위치하도록 레이아웃을 고려해야 합니다.

## 체크박스 & 라디오 버튼 Checkbox & Radio Button 

### 체크박스

- 여러 개의 항목 중 둘 이상의 옵션을 선택해야 할 때 사용합니다.
- 단일 항목일 경우에도 사용할 수 있습니다.
- 보통은 독립적으로 사용하지만, 하위에 있는 여러 항목을 일괄적으로 선택하는 데에 사용하는 상위 체크박스처럼 예외의 경우도 있습니다.
- 컨텍스트에 따라 체크박스만 클릭 가능하게 할지, 체크박스 영역 전체를 활성화할지 고려해야 합니다.

 ![Interpolator](/web/img/checkbox.png)

### 라디오 버튼

- 여러 개의 항목 중 하나의 옵션을 선택해야 할 때 사용합니다.
- 공간의 제약이 없고, 항목의 수를 모두 표시할 수 있을 경우에 적합합니다.
- 항목의 기본값을 선택 상태로 표시합니다.
- 컨텍스트에 따라 라디오 버튼만 클릭 가능하게 할지, 라디오 버튼 영역 전체를 활성화할지 고려해야 합니다.

 ![Interpolator](/web/img/radiobutton.png)

## 셀렉트 Select 

- 여러 개의 항목 중 하나의 옵션을 선택해야 할 때 사용합니다.
- 공간의 제약이 있고, 항목의 수를 모두 표시할 수 없는 경우에 적합합니다.
  ※ 주로 공간의 제약이 있는 모바일에서 사용합니다.
- 항목의 기본값을 선택 상태로 표시합니다.
- Spread 상태는 Native UI로 구현됩니다.

## 텍스트 필드 Text Field 

사용자가 직접 텍스트를 입력하고 편집할 수 있는 컴포넌트입니다. 사용자에게 꼭 필요한 정보만 요청하고, 개인 정보와 관련된 입력 정보는 최소화하여 요청하는 것이 좋습니다.

### 구성

#### **입력 필드 Input Field**

- 시각적으로 정보 입력이 가능한 영역임을 알 수 있도록 해야 하며, 입력 필드의 상태는 서로 명확하게 구분되어야 합니다.

#### **플레이스홀더 placeholder**

- 레이블에 대한 추가 설명을 표시하고, 사용자가 해당 입력 필드에 정보를 입력하면 사라집니다.

#### **레이블 Label**

- 입력 필드에 입력해야 할 정보가 무엇인지 사용자에게 안내합니다. 상황에 따라 플레이스홀더로 대체할 수 있습니다.

#### **입력 텍스트 Input Text**

- 입력 필드에 사용자가 입력한 텍스트입니다. 사용자가 입력한 정보가 잘 입력 되었는지 확인할 수 있도록 가독성이 높아야 합니다.

#### **확인 Validation (optional)**

- 사용자가 텍스트를 입력하는 동안의 상태나 입력 값의 유효성을 아이콘이나 텍스트로 표시하여 시각적인 피드백을 제공합니다.

### 종류

#### Single Line Fields

- 한 줄의 텍스트만 입력할 수 있습니다. (ex. 로그인, 회원가입, 검색)

#### Textarea

- 여러 줄의 텍스트를 입력할 수 있습니다. (ex. 고객센터 > 문의하기)
- 고정된 입력 필드 높이에서 커서가 입력 필드의 맨 아래에 도달하면 스크롤이 발생합니다.

## 모달 Modal 

- 사용자가 페이지 내에서 특정 행동을 했을 때 이에 대한 추가적인 설명이나 결정이 필요한 경우 사용합니다. 이때, 사용자의 컨텍스트를 유지할 수 있도록 하는 것이 중요합니다.
- 사용자가 모달에서 특정 행동을 하지 않아도 원래 화면으로 돌아갈 수 있는 장치를 제공해야 합니다.
  - 닫기 버튼, Escape Key, 모달 창 바깥 영역 클릭
- 웹 뷰어와 관련된 모달(ex. 구매, 대여)은 Theme Color로 지원됩니다.
- 사용자의 작업에 방해가 될 수 있으므로 신중하게 사용합니다.

### 구성

모달의 기본 구성 요소입니다. 상황에 따라 모든 요소가 포함되지 않을 수도 있습니다.Head Part

#### Head Part

-  타이틀과 닫기 버튼이 있습니다. 타이틀은 모달을 띄운 버튼의 레이블과 일치하는 것이 좋습니다. 

#### Body Part

- **머리글 Heading** 
  - 모달 내에서 사용자에게 강력하게 전달하고자 하는 내용을 요약하여 표시합니다. 

- **본문 Text** 
  - 머리글 만으로 전달이 어려운 경우 본문 텍스트를 추가할 수 있습니다. 내용은 짧은 시간 내에 파악할 수 있도록 간결하게 작성합니다.

- **버튼 Button**
  - 버튼은 주로 취소, 확인, 액션 버튼으로 배치됩니다.
  - 버튼 내 레이블은 사용자의 다음 행동을 명확하게 안내할 수 있어야 합니다.
  - 제공하는 버튼의 개수가 많아지지 않도록 주의해야 합니다.

#### **Dim Background**

-  사용자가 모달 창에 집중할 수 있도록 시각적 알림을 제공합니다.

### 사용

#### Modal Basic

- 기본 모달입니다. 사용자의 선택이 필요한 경우나 다음 단계로 넘어가기 위한 중요한 결정이 필요한 경우에 적합합니다. (ex. 웹뷰어 다음 화 보기 > 결제하기)

#### Modal with Text Field

- 사용자가 직접 입력해야 할 텍스트 정보가 있는 경우 Text Field 컴포넌트와 함께 사용합니다. (ex. 회원가입, 로그인, 쿠폰 등록)

#### Modal with Table

- 사용자에게 제공하는 정보나 선택지가 많을 경우 테이블 컴포넌트와 함께 사용합니다. (ex. 리디 셀렉트 > 도서 교체)

### 크기

- 보통 Width 값 기준 300px을 사용하지만, 상황에 따라 더 큰 크기를 사용하기도 합니다.
- 모바일에서는 300px 이상의 크기를 사용할 경우 페이지로 보여주는 것이 좋습니다.

### 위치

- 화면의 크기가 바뀌어도 화면 중앙에 위치할 수 있도록 합니다.
- 필요한 경우 Y축을 고정할 수 있습니다.

## 팝업 Popup 

- 사용자가 즉각적으로 알아야 할 내용이 있거나 중요한 메시지를 전달할 때 시스템 팝업으로 노출합니다.
  - ex. 상세페이지 > 다운로드(앱으로 열기), 구매 목록 도서 영구 삭제

- 사용자의 작업에 방해가 될 수 있으므로 신중하게 사용합니다.

## 탭 Tabs

- 탭은 동일한 페이지 내에서 유사한 정보를 그룹화하여 하나의 행으로 구성할 때 사용합니다. 사용자가 현재 페이지와 관련된 콘텐츠에 쉽고 빠르게 접근할 수 있도록 돕습니다.
- 그룹화 된 탭의 콘텐츠는 사용자가 예측할 수 있도록 서로 관련이 있어야 합니다.
- 탭 레이블은 짧고 간결해야 하며 내용을 적절하게 설명해야 합니다.
  - 사용자가 페이지를 벗어나지 않고 확인할 수 있도록 레이블 뒤에 콘텐츠 관련 숫자를 같이 표기할 수 있습니다.
- 탭의 개수는 모바일 화면을 고려하여 6개 이하로 구성하는 것이 좋습니다.
- 탭의 순서는 사용자의 관점에서 가장 중요한 콘텐츠 순으로 나열합니다.
- 항상 하나의 탭이 활성화된 상태여야 합니다.
  - 기본적으로 해당 페이지 진입 시 첫 번째 탭을 활성화합니다.

- 활성화된 탭은 확실히 강조할 수 있도록 합니다.

### 크기 및 정렬

#### PC

- 탭의 전체 너비는 페이지나 영역에 맞춰 적용합니다.
- 왼쪽을 기준으로 정렬합니다.

#### Mobile

- 전체 Width 값 기준 1/N로 지정합니다.
- 레이블의 길이가 많이 차이 나는 경우에는 Width를 %로 지정할 수 있습니다.

## 테이블 Table 

테이블은 정보를 열과 행으로 구분하여 사용자가 쉽게 읽고 이해할 수 있도록 돕습니다.

### 구성

#### Title Part

- 테이블 내용에 표시할 주요 정보를 명확하고 간결한 레이블로 설명합니다.

####  Data Part

- 타이틀 영역에 대한 상세 정보가 표시됩니다.
- 하나의 행에서 정보를 모두 표시하기 어려운 경우, 해당 정보의 상세페이지를 가질 수 있습니다.
- 상세페이지를 가지고 있는 정보일 경우 Hover 상태 및 아이콘으로 페이지 이동이 가능하다는 표시를 합니다.
- 정보가 많기 때문에 각 정보에 대한 상태 구분이 중요합니다.

### 크기 및 정렬

#### PC

- 테이블의 전체 너비는 페이지나 영역에 맞춰 적용합니다.
- 가격 정보는 우측 정렬 합니다.

#### Mobile

- 정보를 2단으로 구성하여 주요 값은 우측에 정렬합니다.

## 페이지네이션 Pagination

- 페이지네이션은 많은 양의 정보를 불러오기 어려울 때 콘텐츠를 여러 페이지로 나누고, 사용자가 특정 페이지로 탐색/이동할 수 있도록 돕는 역할을 합니다.
- 페이지 하단 중앙에 배치하고, 단일 페이지일 경우에는 노출되지 않습니다.
- 항상 하나의 페이지가 활성화된 상태여야 합니다.
  - 기본적으로 콘텐츠 진입 시, 첫 번째 페이지가 활성화됩니다.

- 사용자가 자신의 위치를 알 수 있도록 활성화된 상태를 확실히 강조해야 합니다.

### 동작

#### 처음/끝 버튼

- **> / 끝** 버튼을 눌렀을 경우 다음 페이지네이션 순서의 가장 처음에 위치한 페이지 번호로 이동합니다.
- **< / 처음** 버튼을 눌렀을 경우 이전 페이지네이션 순서의 가장 처음에 위치한 페이지 번호로 이동합니다.

#### URL Redirection

- URL에서 음수나 한글 입력 시 1 페이지로 자동 리다이렉션 합니다.
- 존재하지 않는 숫자 입력 시 마지막 페이지로 자동 리다이렉션 합니다.

## 토스트 Toast

- 카트에 책을 담거나, 비밀번호를 변경하는 등 사용자 행동에 대한 간단한 피드백을 제공할 때 사용합니다.
- 일시적으로 나타나기 때문에 한눈에 읽히는 메시지로 작성합니다.
- 한번에 하나의 토스트만 표시합니다.
- 모바일에서 하단 Fixed 영역이 있을 경우 Toast와 해당 영역이 겹치지 않도록 간격을 두어 표시합니다.

### 구성

#### **컨테이너 Container**

- 전체 요소를 감싸는 레이어 입니다. 메시지의 가독성을 위해 어두운 배경 컬러를 사용하고, 페이지와 구분할 수 있도록 Shadow를 적용합니다.

#### **아이콘 Icon**

- 토스트의 종류에 따라 관련 아이콘으로 표시합니다.

#### **메시지 Message**

- 사용자에게 전달할 내용이 표시됩니다. 내용은 짧고 명확하게 최대 2줄로 작성합니다.

#### **액션 버튼 Action Button**(optional)

- 메시지와 관련된 페이지 이동이나 작업을 할 수 있도록 텍스트 버튼을 제공합니다. 페이지 이동 시에는 관련 아이콘을 버튼 옆에 배치하여 사용자가 예측할 수 있게 합니다. 

#### **닫기 버튼 Close Button**

- 토스트가 사라지기 전에 사용자가 직접 해당 토스트를 닫을 수 있습니다.

### 종류

상황에 따라 Neutral, Success, Alert 세 가지 종류를 적절하게 사용합니다.

#### Neutral

- 사용자에게 추가적으로 옵션을 제공하거나 일반적인 메시지를 전달할 때 사용합니다. 

### Success

- 작업이 성공적으로 완료되면 사용자에게 알립니다.

### Alert

- 사용자가 작업을 완료하기 전에 해결해야 하는 문제를 알립니다.

### 크기 및 위치

#### PC

- 고정 너비는 400px입니다.
- 화면 상단 영역 중앙에 나타납니다.

#### Mobile

- 좌 우/하단 margin 값 10px로 적용합니다.
- 화면 하단 영역 중앙에 나타납니다.

## 알럿 Alert

페이지 내에 경고 및 안내 메시지를 전달할 경우 사용합니다. 

### 구성

#### **컨테이너 Container**

- 전체 요소를 감싸는 레이어 입니다. 기본적으로 Brown 계열의 BG 컬러를 사용하고, 주변 디자인 요소와 메시지의 목적에 따라 Slategray 계열의 BG 컬러를 사용할 수 있습니다.

#### **아이콘 Icon**

- 경고 및 안내 아이콘을 표시하여 사용자에게 시각적인 알림을 제공합니다.

#### **머리글 Heading**(optional)

- 사용자에게 강력하게 전달하고자 하는 내용을 요약하여 표시합니다. 

#### **본문 Text**

- 머리글 만으로 전달이 어려운 경우에 사용합니다. 내용은 짧은 시간 내에 파악할 수 있도록 간결하게 작성합니다.

#### **버튼 Button**

- 버튼을 배치하여 사용자의 다음 행동을 명확하게 안내합니다.

### 사용

#### 1단

- 주로 Mobile에서 사용합니다.

#### 2단

- 주로 PC에서 사용합니다.
